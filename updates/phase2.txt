### Data Distribution
```sql
SELECT c.name, 
       COUNT(DISTINCT b.id) as batches,
       COUNT(DISTINCT cr.id) as courses,
       COUNT(DISTINCT u.id) as users
FROM coachings c
LEFT JOIN batches b ON c.id = b.coaching_id
LEFT JOIN courses cr ON c.id = cr.coaching_id
LEFT JOIN users u ON c.id = u.coaching_id
GROUP BY c.name;

-- Result:
-- ABC Test Academy      | 2 | 2 | 2
-- Demo Coaching Academy | 2 | 3 | 3
-- XYZ Coaching Center   | 2 | 3 | 0
```

## LESSONS LEARNED

### 1. Schema Namespacing
**Issue:** `auth.is_super_admin()` permission denied  
**Fix:** Create functions in `public` schema  
**Why:** Supabase doesn't allow creating in auth schema

### 2. UUID Quoting
**Issue:** `ERROR: trailing junk after numeric literal`  
**Fix:** Always quote UUIDs: `'uuid-value'`  
**Why:** PostgreSQL tries to parse as scientific notation

### 3. Auth Architecture
**Issue:** Can't insert fake users in seed data  
**Fix:** users.id REFERENCES auth.users(id)  
**Why:** Production-correct design requires real auth  
**Impact:** Users created via Dashboard, not seed SQL
