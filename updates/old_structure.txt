## Key Characteristics

### Single Tenant
- **ONE hardcoded coaching institute:** "ExamEdge Academy"
- No multi-tenancy support
- No coaching_id concept
- Everyone sees same data

### Mock Data
All data stored in local TypeScript files:
- `studentsMockData.ts` - Array of ~10 students
- `batchesMockData.ts` - Array of batches
- `coursesMockData.ts` - Array of courses
- etc.

**Problem:** Static, not scalable, no database

### No Backend
- ❌ No database
- ❌ No API
- ❌ No authentication
- ❌ No user accounts
- ❌ No data persistence

### Single Branding
- Hardcoded name: "ExamEdge Academy"
- Fixed color scheme (Indigo/Purple)
- No customization
- Same theme for all users

### No User Roles
- No login system
- No role-based access control
- Anyone can access admin/student pages

## Why Migration Needed

### Current Limitations
1. **Not scalable** - Can only serve one coaching institute
2. **No real data** - Everything is mock/hardcoded
3. **No persistence** - Refresh = data lost
4. **No SaaS** - Can't sell to multiple customers
5. **No users** - No login, no accounts
6. **No customization** - One size fits all

### Business Impact
- ❌ Can't sell to multiple coaching institutes
- ❌ Can't charge subscription fees
- ❌ Can't customize per customer
- ❌ No recurring revenue model

---

## Migration Goals

**Transform FROM:**
- Single-tenant mock data app
- Hardcoded "ExamEdge Academy"
- No backend, no users

**Transform TO:**
- Multi-tenant SaaS platform
- Unlimited coaching institutes
- Real database (Supabase)
- User authentication
- Custom branding per tenant
- Subscription plans
